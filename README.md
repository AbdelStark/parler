# parler

> Multilingual voice intelligence built on Voxtral. Decision logs from French/English meetings and earnings calls.

**Status: Alpha.** Phase 1 through Phase 7 core surfaces are implemented: typed models, config loading, rendering, orchestration/state handling, audio ingestion, retry utilities, Voxtral transcription, transcript quality evaluation, semantic caching, speaker attribution heuristics, decision extraction, export adapters, canonical CLI commands, and `uv`-based packaging/publishing. Remaining roadmap work is centered on Phase 8 full-system verification, fixture-complete E2E coverage, and later polish.

## The problem

Every meeting produces hours of audio and a pile of "we should really write this up" guilt.

But the bigger problem isn't transcription — it's that transcripts are useless. A 45-minute meeting transcript is 6,000 words of noise. What you actually want is: *what was decided, by whom, with what follow-up*.

Existing tools (Otter.ai, Fireflies, Notion AI, Teams transcription) all share the same two failures:
1. **English-first**: French, German, Spanish, Italian — and code-switching between them — produce mediocre results
2. **Summary, not decisions**: they give you "the team discussed X" instead of "the team decided X; @Marie owns it by Friday"

`parler` (French: *to speak*) is built on Voxtral — Mistral's multilingual voice model — and is designed for European teams who actually work in European languages. Its output is not a transcript or a summary. It's a **Decision Log**: a structured record of what was agreed, rejected, assigned, and left open.

parler process meeting.mp3 --lang fr,en

## Current implemented surface

```bash
uv python install
uv sync --locked --group dev

# Validate config loading
uv run parler config validate --config parler.toml

# Inspect the CLI
uv run parler --help
uv run parler transcribe meeting.mp3 --format json
uv run parler report --from-state .parler-state.json --format html
uv run parler cache list

# Run the implemented test-backed slice
uv run pytest \
  tests/unit/test_config_loading.py \
  tests/unit/test_cli_commands.py \
  tests/unit/test_report_rendering.py \
  tests/unit/test_pipeline_orchestration.py \
  tests/unit/test_audio_ingestion.py \
  tests/unit/test_chunk_assembly.py \
  tests/unit/test_transcript_quality.py \
  tests/unit/test_speaker_attribution.py \
  tests/unit/test_decision_extraction_parsing.py \
  tests/unit/test_deadline_resolution.py \
  tests/unit/test_deadline_resolution_parametrized.py \
  tests/integration/test_retry_behavior.py \
  tests/integration/test_voxtral_integration.py \
  tests/integration/test_cache_behavior.py \
  tests/integration/test_mistral_extraction.py \
  tests/integration/test_export_integrations.py \
  tests/property/test_deadline_resolver_properties.py \
  tests/property/test_parsing_properties.py -q

# Build distributable artifacts
uv build
```

Decision extraction, canonical Markdown/HTML/JSON rendering, isolated Notion/Linear/Jira/Slack export adapters, checkpoint loading, and the canonical `process` / `transcribe` / `extract --from-state` / `report --from-state` / `cache` CLI commands are now implemented and test-backed. The next major roadmap focus is Phase 8 full-system verification with provisioned E2E fixtures and benchmark baselines.

## Stack

- **Transcription**: Voxtral (Mistral's multilingual voice model) via La Plateforme API
- **Decision extraction**: mistral-large-latest via La Plateforme API
- **Audio preprocessing**: FFmpeg (optional, for format conversion and chunking)
- **Language**: Python 3.11+
- **Packaging / workflow**: `uv`, `uv_build`, GitHub Actions Trusted Publishing
